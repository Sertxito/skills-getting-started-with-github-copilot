# Planning Readiness Checklist

Use this checklist before implementation starts.

## Scope and Intent

- [ ] Business objective is clearly stated.
- [ ] Out-of-scope items are explicitly listed.
- [ ] Success criteria are measurable.

## Requirements Quality

- [ ] Acceptance criteria are testable and unambiguous.
- [ ] User impact and edge cases are documented.
- [ ] Dependencies and assumptions are listed.

## Ownership and Collaboration

- [ ] Role owner is assigned for delivery coordination.
- [ ] Quality owner is assigned for test strategy.
- [ ] Stakeholder communication owner is assigned.

## Technical and Validation Plan

- [ ] Implementation approach is documented.
- [ ] Test strategy is defined (unit/integration/regression where applicable).
- [ ] Observability or verification approach is defined.

## Delivery and Risk

- [ ] Risks are listed with mitigation actions.
- [ ] Timeline and checkpoints are realistic.
- [ ] Escalation path is documented for blockers.

## Release Preparation

- [ ] Rollout strategy is defined.
- [ ] Rollback conditions are defined.
- [ ] Release communication plan is prepared.
