# Release Readiness Checklist

Use this checklist before production deployment.

## Build and Test Health

- [ ] Relevant automated tests passed.
- [ ] High-priority defects are resolved or explicitly deferred with approval.
- [ ] Required manual checks are completed.

## Documentation and Traceability

- [ ] PR description summarizes changes and risks.
- [ ] PR links to the source issue.
- [ ] User-facing or operational notes are documented.

## Operational Safety

- [ ] Rollback plan is validated.
- [ ] Monitoring/alerts for affected areas are confirmed.
- [ ] On-call or support contacts are informed.

## Communication and Approvals

- [ ] Stakeholders have release timing and impact summary.
- [ ] Required approvers have signed off.
- [ ] Go/no-go decision is recorded.

## Post-Release Checks

- [ ] Smoke checks are completed after deployment.
- [ ] Metrics and error rates are reviewed.
- [ ] Follow-up actions are logged for next iteration.
